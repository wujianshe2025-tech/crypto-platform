## 🌐 部署到Vercel

### 方法1: 通过Vercel CLI

1. 安装Vercel CLI（如果还没安装）
```bash
npm install -g vercel
```

2. 登录Vercel
```bash
vercel login
```

3. 部署
```bash
cd crypto-platform/client
vercel
```

4. 按照提示操作：
   - 选择项目
   - 确认设置
   - 等待部署完成

### 方法2: 通过Vercel网站

1. 访问 https://vercel.com
2. 登录你的账号
3. 点击 "Add New Project"
4. 选择你的GitHub仓库
5. 配置项目：
   - Framework Preset: Vite
   - Root Directory: `client`
   - Build Command: `npm run build`
   - Output Directory: `dist`
6. 点击 "Deploy"

## ⚙️ Vercel配置

如果需要，创建 `vercel.json` 文件：

```json
{
  "buildCommand": "npm run build",
  "outputDirectory": "dist",
  "framework": "vite",
  "rewrites": [
    {
      "source": "/(.*)",
      "destination": "/index.html"
    }
  ]
}
```

## 🔧 环境变量（可选）

如果将来需要配置API密钥，在Vercel项目设置中添加环境变量：

1. 进入项目设置
2. 选择 "Environment Variables"
3. 添加变量（例如：`VITE_API_KEY`）
4. 重新部署

在代码中使用：
```typescript
const apiKey = import.meta.env.VITE_API_KEY;
```

## 📝 注意事项

1. **API限制**: 
   - CoinGecko: 每分钟10-50次请求
   - CryptoCompare: 每秒20次请求

2. **CORS**: 
   - 这些API支持跨域请求
   - 无需额外配置

3. **缓存**: 
   - 考虑添加缓存机制减少API调用
   - 可以使用localStorage或React Query

4. **错误处理**: 
   - 已包含基本错误处理
   - 生产环境建议添加错误监控

## 📞 问题排查

### 构建失败
- 检查依赖是否完整安装

- 确认Node版本（建议18+）

### 页面空白

- 检查API是否可访问

### API错误
- 检查网络连接
- 确认API服务状态
- 查看请求频率是否超限
